# Mount — Documentation Pack

Hi — these are the written reports for the Mount project. Read in any order, but if you only have a few minutes I'd suggest:

1. `PROJECT_OVERVIEW.md` (5 min) — what the app is and how the pieces fit together.
2. `README.md` (3 min) — how to run it locally.
3. `DESIGN_DECISIONS.md` (5 min) — the choices that shaped the codebase.

For the full picture:

| File | What's inside |
|------|---------------|
| `README.md` | Setup instructions, env vars, common problems. |
| `PROJECT_OVERVIEW.md` | Architecture, request lifecycle, folder layout, how the dashboard panel is computed. |
| `API.md` | Every endpoint, with request/response shape and error codes. |
| `DATABASE_SCHEMA.md` | The seven Prisma models, with notes on relationships and indexes. |
| `DESIGN_DECISIONS.md` | The non-obvious choices and the things I gave up. |
| `SECURITY_NOTES.md` | What's in place, the known gaps, and the order I'd fix them. |
| `AI_USAGE.md` | An honest note on where AI was and wasn't used. |
| `WHAT_NEXT.md` | The roadmap if I had another week or two. |

— happy to walk through any of this in person.
